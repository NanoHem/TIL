핵심 쏙쏙 Git - 2주차

머리말
2주차에서는 협업하기 위한 Git 기본 개념을 익힌고 두 명 이상과 협업하는 Git 프로젝트를 만들수 있고, 기능별로 나누어 작업내역을 남길 수 있는 것을 배운다.

내용
Issue할당을 통해서 누가 이 작업 할 것인지 정하는 것이다. 그리고 브랜치(Branch)로 넘어와 각자 맡은 것을 작업하고 머지(merge)를 이용하여 합친다. 브랜치를 이용해서 만든 파일은 예를 들어서 타임라인이라고 생각된다. 하나의 타임라인에서 무엇인가 추가하게 되거나 수정하게될때 옆에 선을 긋고 수정 또는 추가를 한다. 그리고 머지(merge)는 다시 원래의 타임라인으로 돌아오는 것이라고 이해하였다. 

소감
강의를 들으면서 직접 충돌을 일으켜 수정하는 방법을 가르쳐주어서 감사했습니다. 아마도 나중에 프로젝트를 할때 분명 충동이 발생할 것이라고 생각합니다. 미리 알려주어서 감사합니다.

ErrorNote
에러가 발생하지 않아서 쓸게 없습니다. ㅎㅎ

2주차 끝!
